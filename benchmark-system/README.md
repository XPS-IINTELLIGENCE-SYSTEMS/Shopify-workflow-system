# Benchmark System

Universal module for reverse engineering and forward engineering any public system, website, funnel, app, offer, business model, workflow, or competitor pattern.

## Purpose

The Benchmark System takes any reference target and decomposes it exhaustively.

It answers:

- What is this system?
- What does it do?
- How does it work?
- What parts does it contain?
- What does each part do?
- What category does each part belong to?
- Why does that part belong in that category?
- What other parts does it connect to?
- What happens if that part is missing?
- What is the visual design, layout, color, shape, copy, CTA, data flow, and business purpose?
- What is missing?
- What can be improved?
- How do we forward-engineer a better version?

- AI tool
- Local business
- Partner website
- Competitor website
- Internal workflow

## Rule

Benchmarking is for analysis, decomposition, structure, and improvement.

Do not copy protected branding, private code, copyrighted copy, restricted data, impersonation elements, or confidential systems.

## Workflow

```txt
Benchmark target provided
↓
Preflight intake
↓
Scrape / ingest / normalize public data
↓
Exhaustive component inventory
↓
Category + connection map
↓
Missing part analysis
↓
Failure impact analysis
↓
Improvement matrix
↓
Forward-engineering specification
↓
Shopify Workflow build packet
```

## Required Output

Every benchmark must produce:

- Executive summary
- Full system description
- Component inventory
- Visual inventory
- Funnel map
- Offer map
- CTA map
- Data/input/output map
- User journey map
- Backend/automation inference map
- Content/SEO map
- Trust/proof map
- Monetization map
- Risk/compliance map
- Missing parts list
- Improvement matrix
- Better-system blueprint
- Forward-engineering build spec
- Shopify Workflow build requirements
