# Benchmark Reverse + Forward Engineering Module

Purpose:
- Start with an existing system, website, funnel, app, tool, or competitor.
- Reverse engineer what it is, how it works, what it offers, what it is missing, and why users buy or use it.
- Identify gaps, weaknesses, risks, limitations, and improvement opportunities.
- Forward engineer a better system through Shopify Workflow.

## Workflow

```txt
Target system URL / app / competitor
↓
Preflight benchmark intake
↓
Scrape / ingest / normalize
↓
Capability map
↓
Gap map
↓
Improvement matrix
↓
Forward engineering spec
↓
Shopify / app / funnel build packet
↓
QA + approval gate
```

## Output Tables

- Benchmark Targets
- Feature Inventory
- Offer/CTA Map
- Funnel Map
- Pricing/Monetization Map
- Content/SEO Map
- UX/UI Gap Map
- Automation Gap Map
- Compliance/Risk Map
- Improvement Matrix
- Forward Engineering Spec

## Rule

Do not copy protected branding, code, copyrighted content, or private systems.
Benchmark for structure, market logic, UX patterns, offer strategy, and improvement opportunities.

## Position in Shopify Workflow

This module runs before market discovery and before build planning when the operator says:

- build something like this
- clone the model but make it better
- reverse engineer this system
- benchmark this competitor
- improve this existing workflow
